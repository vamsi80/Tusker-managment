# Build Error Fix Plan

## Summary

The build fails due to multiple issues:

1. Missing `react-icons` package in dependencies
2. Server-only modules being imported in client components (server-only, next/headers, nodemailer)

## Root Causes

### Issue 1: Missing `react-icons` package

- Multiple components import from `react-icons/fa` but the package isn't installed
- Affected files: `loginForm.tsx`, `signUpForm.tsx`, `workspace-info-form.tsx`, `team-member-columns.tsx`, etc.

### Issue 2: Server-only imports leaking to client

The main chain is:

1. `quick-create-subtask.tsx` (Client Component) imports `getWorkspaceTaskCreationData`
2. `get-workspace-task-creation-data.ts` imports `requireUser` from `@/lib/auth/require-user`
3. `require-user.ts` imports `auth` from `@/lib/auth` (has `server-only`)
4. `get-workspace-task-creation-data.ts` imports `WorkspaceService` (imports `revalidateTag`)
5. `lib/email.ts` (imported by auth.ts) uses `nodemailer` which has Node.js-only imports

## Fix Steps

### Step 1: Install missing dependencies

- Add `react-icons` to package.json

### Step 2: Fix server-only leakage (CRITICAL)

The data function `getWorkspaceTaskCreationData` is being called from a Client Component, but it imports server-only code. Two solutions:

**Solution A (Recommended):** Make the data fetching happen in a Server Component and pass data as props to Client Component

**Solution B:** Create a client-safe API route wrapper

Given the architecture, Solution A is cleaner. The fix involves:

1. Modify `quick-create-subtask.tsx` to:
   - Accept `WorkspaceTaskCreationData` as a prop instead of fetching it client-side
   - Move the data fetching to a parent Server Component

2. Alternatively, create a Server Action that fetches the data (but this also needs server-only handling)

Actually, the simplest fix is:

- Change `getWorkspaceTaskCreationData` to NOT import `requireUser` directly at module level
- Instead, pass the user from the calling context

### Step 3: Fix `server-only` package not found

This is likely due to missing or misconfigured `server-only` package. The import is there but might need verification.

### Step 4: Fix nodemailer Node.js imports

`nodemailer` uses Node.js built-in modules (fs, dns, net, etc.) which don't exist in the browser. The key fix is ensuring `lib/auth.ts` (and any file that imports it) is only used in Server Components/Server Actions.

## Implementation

1. Add `react-icons` to package.json
2. Create a client-safe data fetching approach - change how `QuickCreateSubTask` fetches its data:
   - Either use a Server Action that returns only serializable data
   - Or pass data from a parent Server Component

3. Ensure all files using `server-only`, `next/headers`, `next/cache` are only imported in Server Components
4. Verify all parent component chains are server-only compatible
